# Corta как сайт (без Telegram-бота)

Приложение работает в **обычном браузере** и как **PWA** (можно «Добавить на экран»).

## Что уже работает без бота

- Весь тест (слова, фланкер, реакция, лица, Струп)
- Результат и расширенный отчёт
- История прохождений (в браузере)
- Заявка на консультацию (email → hello@bookvolon.ru)

## Оплата (Робокасса)

Пока Робокасса не подключена, при нажатии «Оплатить» показывается сообщение «Оплата подключается».

После подключения в Amvera → **Запуск**:

```env
PAYMENT_PROVIDER=robokassa
ROBOKASSA_MERCHANT_LOGIN=...
ROBOKASSA_PASSWORD1=...
ROBOKASSA_PASSWORD2=...
ROBOKASSA_IS_TEST=1
PAYMENTS_PUBLIC_BASE_URL=https://ваш-домен.amvera.io
```

В кабинете Робокассы (**Технические настройки**):

| Поле | Значение | Метод |
|------|----------|--------|
| **Result URL** | `https://cortaapp.ru/robokassa/result` | POST |
| **Success URL** | `https://cortaapp.ru/` | **GET** (без `?` и параметров!) |
| **Fail URL** | `https://cortaapp.ru/` | **GET** (без `?` и параметров!) |

Не указывайте `?robokassa=success` в Success URL — Робокасса выдаст ошибку валидации.  
`sessionId` и продукт передаются через `Shp_sessionId` / `Shp_product` при создании счёта; после оплаты Робокасса сама допишет их в query.

Пересоберите проект после смены переменных.

## Домен

Откройте в браузере тот же URL, что на Amvera, например  
`https://corta-ns-1234dinara.amvera.io`

Telegram Mini App по-прежнему может работать параллельно, если бот открывает тот же URL.

## Проверка

1. Откройте сайт в Chrome/Safari (не Telegram).
2. Пройдите тест до экрана оплаты.
3. После подключения Робокассы — оплата уводит на страницу банка и возвращает на сайт.
